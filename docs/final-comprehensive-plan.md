# 1. 서비스 개요

## 1-1. 서비스 비전

> **"누구나 인스타 크리에이터처럼 찍을 수 있게 해주는 포토존 플랫폼"**
>
> **스냅스타 (SnapStar)** — 스냅 촬영의 스타가 되다

인스타 사진들이 지도 위에 펼쳐져 있고, 마음에 드는 사진을 누르면
어디서 어떻게 찍었는지 알려주고, 현장에서는 카메라 앱이 자동으로 세팅해준다.
최소한의 노력으로 최대 퀄리티의 사진을 소셜에 바로 포스팅한다.

## 1-2. 서비스 구조: 메인(스냅스타 웹) + 서브(카메라 앱)

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  🗺️ 메인 서비스 = 스냅스타 웹 (SnapStar Web)                  │
│  ──────────────────────────────                              │
│                                                             │
│  [1층] 인스타 사진 + 2D 맵 (진입점)                          │
│  · 지도 위에 인스타 사진 썸네일이 포토존 마커로 배치           │
│  · 사진을 보고 끌리는 곳을 탐색 ("이거 어디서 찍은 거야?")    │
│  · 포토존 클릭 → 촬영 정보 + 디렉션 안내                     │
│  · 여행 콘텐츠 (숙소, 음식, 여행지) 연결                     │
│  · 길찾기 (포토존·여행지 간 이동 경로)                       │
│                                                             │
│  [2층] MapLibre 3D (촬영 디렉션 보조)                        │
│  · 2D 맵에서 "촬영 방향 보기" 시 활성화                       │
│  · 건물 높이, 지형으로 촬영 위치·방향·거리감 파악              │
│  · 3D는 정보 전달 보조 도구 (메인 경험이 아님)                │
│                                                             │
│  [3층] 고퀄 3D 뷰 (별도 화면 전환)                           │
│  · "이 장소를 3D로 보기" 선택 시 별도 뷰어로 전환             │
│  · 촬영자 위치 + 피사체(연인, 사물) 배치를 3D로 표현          │
│  · Three.js / 가우시안 스플래팅 / 포토그래메트리               │
│  · 촬영 구도를 입체적으로 이해할 수 있는 고품질 경험           │
│                                                             │
│          │  포토존 선택 → 길찾기 → 현장 도착                  │
│          ▼                                                  │
│                                                             │
│  📷 서브 서비스 = 카메라 앱 (모바일 별도 앱)                  │
│  ──────────────────────────────────────────                  │
│  · 포토존 도착 시 프리셋 자동 적용 (화각, 노출, 색감)         │
│  · 촬영 가이드 (구도, 피사체 위치, 앵글 오버레이)             │
│  · 자동 보정 → 인스타/소셜에 바로 포스팅                     │
│  · 추후 영상 촬영 가이드 확장                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 1-3. 사용자 여정 전체 흐름

```
[발견]              [이해]             [계획]          [이동]        [촬영]         [공유]

2D 맵 위의         포토존 클릭 →      여행 콘텐츠      길찾기로      카메라 앱      자동 보정
인스타 사진에       촬영 위치/방향     (숙소/음식)      현장 도착     프리셋 적용    소셜 포스팅
끌려서 탐색        + 디렉션 안내      확인                          가이드 촬영
                   (3D 보조)
                   (고퀄 3D 선택)

◀─── 1층: 2D맵+사진 ──▶◀── 2층/3층: 3D ──▶◀────────▶ ◀── 서브: 카메라 앱 ──▶
```

## 1-4. 3D의 역할 재정의

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ❌ 기존 접근: 3D 지도가 메인 경험                            │
│     → 3D에서 시작, 3D 안에서 모든 것을 해결                   │
│     → 문제: 3D 건물이 투박해서 사진과 안 비슷함               │
│                                                             │
│  ✅ 새 접근: 인스타 사진이 메인, 3D는 보조                    │
│                                                             │
│     인스타 사진 + 2D 맵  =  메인 경험 (발견·탐색)             │
│     MapLibre 3D          =  촬영 디렉션 보조 (방향·거리)      │
│     고퀄 3D (별도 화면)   =  촬영 구도 이해 (촬영자+피사체)    │
│                                                             │
│  3D 지도는 인공위성 사진 vs 벡터 지도처럼                     │
│  "추상화된 정보 전달"이 목적이지 "실사 재현"이 아니다          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 1-5. 문제 → 해결

| 사용자 Pain Point | 서비스 해결 |
|---|---|
| 인스타에서 예쁜 사진을 봤는데 어디서 찍은 건지 모름 | **지도 위에 인스타 사진이 직접 배치**되어 바로 위치 파악 |
| 사진은 봤는데 어떤 방향/앵글인지 감이 안 옴 | 촬영 위치·방향을 2D 맵에 표시 + **3D로 디렉션 보조** |
| 현장에서 촬영자·피사체를 어디에 세워야 할지 모름 | **고퀄 3D 뷰**에서 촬영자+피사체 배치를 입체적으로 확인 |
| 카메라 세팅을 몰라서 크리에이터처럼 못 찍음 | 카메라 앱이 프리셋+가이드 자동 적용 |
| 보정이 어려워서 포스팅까지 시간이 오래 걸림 | 자동 보정 → 소셜에 바로 포스팅 |
| 포토존 주변 맛집·숙소 정보가 따로 없음 | 포토존 → 여행 콘텐츠가 자연스럽게 연결 |

## 1-6. 차별화 포인트

| 기존 서비스 | 이 서비스 |
|---|---|
| 구글맵에 핀만 찍힘, 사진 없음 | **인스타 사진이 지도 위에 직접 배치** |
| 위치만 알려줌, 방향은 모름 | 촬영 방향+앵글을 **2D 화살표 + 3D 보조**로 안내 |
| 촬영자·피사체 위치를 모름 | **고퀄 3D에서 촬영자+피사체 배치**를 입체 확인 |
| 위치만 알려줌, 찍는 법은 모름 | **프리셋+촬영 가이드+자동 보정**까지 원스톱 |
| 사진 정보와 여행 정보가 분리 | 포토존에서 **숙소·음식·여행지로 자연스럽게 연결** |

## 1-7. 핵심 UX (5-Beat)

```
[Beat 1]  2D 맵 위의 인스타 사진 썸네일을 탐색
          → "이 사진 예쁘다, 어디지?" 호기심으로 클릭
              ↓
[Beat 2]  포토존 상세: 촬영 위치·방향·팁 + 인스타 원본
          → 2D 맵에 촬영 방향 화살표 + MapLibre 3D로 디렉션 보조
          → (선택) "3D로 보기" → 별도 고퀄 3D 뷰어에서
             촬영자 위치 + 피사체 배치를 입체적으로 확인
              ↓
[Beat 3]  여행 계획: 주변 숙소/음식/여행지 + 길찾기
              ↓
[Beat 4]  현장 도착 → 카메라 앱 실행
          → 프리셋 자동 적용 + 촬영 가이드 오버레이        ← 서브 앱
              ↓
[Beat 5]  가이드대로 촬영 → 자동 보정
          → 인스타/소셜에 바로 포스팅                      ← 서브 앱
```

**MVP 범위:** Beat 1~2 (인스타 사진 2D 맵 + 촬영 디렉션)
**확장:** Beat 2 고퀄 3D, Beat 3 여행 콘텐츠, Beat 4~5 카메라 앱

# 2. 기술 스택 & 자료 조사

## 2-1. 3D 지도 전략: MapLibre + 포토존별 고퀄 3D (하이브리드)

### 핵심 전략 변경

```
기존: CesiumJS로 전 지구 3D 건물을 보여주는 접근
     → 문제: OSM Buildings가 박스형으로 투박함

변경: MapLibre GL JS (2D/2.5D 지도) + 포토존별 고퀄 3D 뷰어
     → 지도는 "정보 전달 도구" 역할에 충실 (위치·방향·동선)
     → 특정 포토존만 Three.js/가우시안 스플래팅으로 고퀄 3D 제공
```

### 왜 MapLibre인가

| 장점 | 상세 |
|------|------|
| **완전 무료** | BSD-2 라이선스, 벤더 종속 없음 |
| **Three.js 네이티브 통합** | 커스텀 레이어로 3D 모델/가우시안 스플래팅 직접 임베드 |
| **카메라 제어** | flyTo, easeTo로 bearing/pitch/zoom 부드러운 전환 |
| **3D 건물** | fill-extrusion으로 OSM 건물 높이 돌출 |
| **3D 지형** | DEM 소스로 실제 고도 표현 |
| **지도 스타일 자유** | 완전 커스터마이징 가능 |
| **웹 개발 친화** | React/Vue/Angular 공식 바인딩 |
| **활발한 개발** | GitHub ★7.5k, 매주 릴리스 |

### 하이브리드 구조

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  [MapLibre GL JS 지도]  ←  기본 뷰                           │
│                                                             │
│  · 2D/2.5D 벡터 지도 (pitch 최대 85°)                       │
│  · 포토존 마커 7곳 표시                                      │
│  · flyTo로 포토존 간 이동                                    │
│  · fill-extrusion으로 건물 높이 표시                          │
│  · 동선·거리 한눈에 파악                                     │
│                                                             │
│       │ 포토존 마커 클릭                                     │
│       ▼                                                     │
│                                                             │
│  [포토존 전용 뷰] ← Three.js 커스텀 레이어 또는 전체화면 뷰어  │
│                                                             │
│  · 방식 A: Street View 360° (즉시 가능, 무료)               │

│                                                             │
│  + 인스타 사진 나란히 비교 패널                               │
│  + 촬영 팁, 추천 시간, 혼잡도 정보                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

# 3. MVP 구현 시나리오 (도쿄타워)

## 3-1. MVP 범위

- 랜드마크: 도쿄타워 1곳
- 포토존: 7곳 (운영자 수동 등록)
- 콘텐츠: 각 포토존당 인스타 1개 + 유튜브 1개
- 기능:
  - 인스타 사진 썸네일이 2D 맵 위에 마커로 배치 (진입점)
  - 포토존 클릭 → 촬영 위치·방향 안내 + 콘텐츠 패널
  - MapLibre 3D 디렉션 보조 (pitch 전환)
  - 투어 모드 (포토존 순차 슬라이드)

## 3-3. 개발 일정 (1인 기준)

| Phase | 기간 | 작업 |
|-------|------|------|

## 7-3. 포토존별 Three.js 커스텀 레이어 (고퀄 3D)

```javascript
import * as THREE from 'three';
import { GLTFLoader } from 'three/addons/loaders/GLTFLoader.js';

// MapLibre 커스텀 레이어로 Three.js 3D 모델 삽입
const customLayer = {
  id: 'photozone-3d',
  type: 'custom',
  renderingMode: '3d',

  onAdd(map, gl) {
    this.camera = new THREE.Camera();
    this.scene = new THREE.Scene();
    this.renderer = new THREE.WebGLRenderer({ canvas: map.getCanvas(), context: gl });
    this.renderer.autoClear = false;

    // 포토존 위치에 glTF 모델 로드
    const loader = new GLTFLoader();
    loader.load('/models/tofuya-stairs.glb', (gltf) => {
      this.scene.add(gltf.scene);
    });
  },

  render(gl, matrix) {
    this.renderer.resetState();
    this.renderer.render(this.scene, this.camera);
    map.triggerRepaint();
  },
};

map.on('load', () => {
  map.addLayer(customLayer);
});

// 촬영 위치 → 랜드마크 방위각 자동 계산
function calculateHeading(shooterLat, shooterLng, towerLat, towerLng) {
  const dLng = toRadians(towerLng - shooterLng);
  const lat1 = toRadians(shooterLat);
  const lat2 = toRadians(towerLat);
  const y = Math.sin(dLng) * Math.cos(lat2);
  const x = Math.cos(lat1) * Math.sin(lat2)
          - Math.sin(lat1) * Math.cos(lat2) * Math.cos(dLng);
  return (toDegrees(Math.atan2(y, x)) + 360) % 360;
}
```

# 8. 포토존 확장 — 세 가지 추가 경로

## 8-1. 경로 A — AI 자동 수집 (v1.5)

```
사진 업로드 → [AI] EXIF GPS 추출 또는 AI 지오로케이션
           → [AI] heading 자동 계산 (촬영위치→타워 방위각)
           → [AI] pitch/fov 1차 추정
           → [3D 프리뷰 생성] 원본 사진과 나란히 비교
           → [제작자] 슬라이더로 미세 보정 ← ★ AI 오차 보정
           → [검증] 유사도 > 60% → 자동승인 / < 60% → 운영자 검수 큐
           → 게시
```

**AI가 하는 것:** GPS, heading (80% 수준으로 빠르게)
**제작자가 하는 것:** pitch/fov 미세조정 (3D 프리뷰 보면서 슬라이더 조작)
**구조: "AI 초안 + 사람 마감"**

### AI 보조 등록 시 속도 향상

```
v1.0 (100% 수동):  사진→위치→핀→heading→pitch→fov   = 1건당 20~30분
v1.5 (AI 보조):    사진→[자동]→[자동]→[자동]→미세조정 = 1건당 3~5분 (6배↑)
```

## 8-2. 경로 B — 운영자 직접 추가 (v1.0~)

관리자 CMS에서 3D 지도 위에 핀 드롭 → 슬라이더로 heading/pitch/fov 직접 세팅
→ 참고 사진과 3D 프리뷰를 나란히 보며 조정 → 메타 정보 입력 → 게시

MVP에서 바로 사용하는 기본 경로.

## 8-3. 경로 C — 사용자 UGC (v2.0+)

```
사용자: 사진 업로드 → GPS 자동추출 → 지도에서 위치 확인
      → 3D 뷰 슬라이더로 앵글 맞추기 → 제출

시스템: 자동 필터 (NSFW, 범위 체크, 중복 체크)
      → AI 보정 (heading 자동, 유사도 스코어)
      → 유사도 높으면 자동승인 / 낮으면 운영자 검수
      → 게시 → 제출자에게 알림 + 기여 배지
```

## 10-8. 성장 플라이휠

```

     ↓

     ↓
다른 도시 랜드마크로 확장 → 같은 플라이휠 반복
```
