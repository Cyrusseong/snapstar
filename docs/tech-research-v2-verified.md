# 4. v1.5 AI 포토존 수집

## 4-1. Apple Depth Pro — FOV 자동 추정 ★핵심

| 항목 | 내용 |
|------|------|
| HF 모델 | `apple/DepthPro-hf` |
| HF 문서 | https://huggingface.co/docs/transformers/en/model_doc/depth_pro |
| GitHub | https://github.com/apple/ml-depth-pro |
| 릴리스 | 2024-10-02 발표, 2025-02-10 HF Transformers 공식 통합 |
| 마지막 업데이트 | 2025-08 (컬렉션), 2025-03 (모델 파일) |
| 크기 | ~1GB |
| M4 Mac | ✅ PyTorch MPS 지원 |
| 출력 | **미터 단위 깊이맵 + FOV(화각) + focal_length** |
| 라이선스 | Apple Sample Code License |

**우리 서비스에서의 역할:**
```
사진 1장 → Depth Pro → { fov: 35.2°, focal_length: 1280px, depth_map: HxW }
                             │                │
                             ▼                ▼
                    3D 카메라 fov 세팅   타워까지 거리 추정
```

**사용 코드:**
```python
from transformers import DepthProImageProcessorFast, DepthProForDepthEstimation
import torch

device = "mps"  # M4 Mac
model = DepthProForDepthEstimation.from_pretrained("apple/DepthPro-hf").to(device)
processor = DepthProImageProcessorFast.from_pretrained("apple/DepthPro-hf")

inputs = processor(images=image, return_tensors="pt").to(device)
with torch.no_grad():
    outputs = model(**inputs)

result = processor.post_process_depth_estimation(outputs, target_sizes=[(h, w)])
fov = result[0]["field_of_view"]          # ★ FOV 자동 추정
focal = result[0]["focal_length"]         # focal length (px)
depth = result[0]["predicted_depth"]      # 미터 단위 깊이맵
```

# 10. 가우시안 스플래팅 — 실사 공간 토글 기능 (v2.0+)

## 9-1. 3D 지도와 가우시안 스플래팅의 역할 구분

**핵심 원칙: 3D 지도는 "정보 전달 도구"이지 "실사 재현"이 아니다**

인공위성 사진보다 벡터 지도가 정보 습득과 판독에 빠른 것처럼,
3D 지도는 추상화된 형태로 위치·방향·거리·동선을 전달하는 것이 목적이다.
인스타 사진과 "비슷하게 보이는 것"은 3D 지도의 역할이 아니다.

가우시안 스플래팅은 **특정 공간을 실제 오브젝트로 체험**하고 싶을 때,
토글로 전환하는 별도 기능이다.

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  3D 지도 (기본 상태)                                      │
│  ═══════════════════                                     │
│  · 추상화된 박스형 건물                                    │
│  · 위치/방향/거리 정보를 빠르게 전달                        │
│  · 포토존 마커, 카메라 시점 전환, 동선 파악                 │
│  · 지도 본연의 역할에 충실                                 │
│                                                          │
│          ↕ [🔄 실제 모습 보기] 토글                        │
│                                                          │
│  가우시안 스플래팅 (토글 ON 시)                             │
│  ═══════════════════════════                              │
│  · 해당 포토존 주변 ~50m만 실사 3D로 전환                  │
│  · "이 공간이 실제로 어떻게 생겼는지" 체험                  │
│  · 전체 지도를 실사로 바꾸는 것이 아님 (국소적)             │
│  · 토글 OFF하면 다시 추상 3D 지도로 복귀                   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 9-2. 적용 가능한 기술

| 기술 | 링크 | 상태 | 역할 | M4 Mac |
|------|------|------|------|--------|
| **Apple SHARP** | [apple/Sharp (HF)](https://huggingface.co/apple/Sharp) | 🟢 Apple 관리 | 사진 1장 → 즉석 3DGS 생성 (1초 이내) | ⚠️ 예측 MPS 가능, 렌더링 CUDA |
| **GaussianSplats3D** | [GitHub](https://github.com/mkkellogg/GaussianSplats3D) | 🟢 활발 | Three.js 기반 웹 3DGS 뷰어 | ✅ 브라우저 WebGL |
| **SuperSplat** | [PlayCanvas](https://playcanvas.com/supersplat/editor) | 🟢 활발 | 브라우저 기반 3DGS 편집·압축 도구 | ✅ 브라우저 |
| **Splatt3R** | [HF Space](https://huggingface.co/spaces/brandonsmart/splatt3r) | 🟢 | 사진 1~2장 → 3DGS | ⚠️ GPU 권장 |
| **COLMAP + 3DGS 원본** | [GitHub](https://github.com/graphdeco-inria/gaussian-splatting) | 🟢 SIGGRAPH 2023 | 다중 사진 → 정밀 3DGS 학습 | ❌ CUDA 필수 |
| **SPZ 압축 (Niantic)** | 오픈소스 | 🟢 | .ply → ~10배 압축 (웹 전송용) | ✅ |

### 파일 포맷 & 압축

```
원본 .ply (수백 MB)
    │
    ├── SPZ 압축 (Niantic) ───→ ~10배 축소
    ├── SOG 압축 (PlayCanvas) ──→ ~15~20배 축소
    └── glTF KHR_gaussian_splatting ──→ 2025-08 Khronos 표준화 진행중
```

## 9-3. 제약사항

| 제약 | 상세 | 우회 방법 |
|------|------|----------|
| **3DGS 학습에 CUDA GPU 필요** | RTX 3090/4090 권장 (24GB VRAM) | 클라우드 GPU (Colab 무료, Lambda Cloud) 또는 Apple SHARP(단일 사진) |
| **M4 Mac에서 학습 불가** | Apple Silicon은 CUDA 미지원 | 뷰어만 로컬, 학습은 클라우드 |
| **야외 대규모 씬 한계** | 하늘·반사면·유리에서 아티팩트 | 포토존 반경 ~50m 소규모 씬으로 제한 |
| **도쿄타워 자체는 도전적** | 반사 금속 + 대규모 구조 | 타워 "주변 공간"을 스캔, 타워 자체는 포함하되 완벽하지 않을 수 있음 |
| **파일 크기** | 원본 수백 MB | SPZ/SOG 압축 필수 (웹 전송 ~10~50MB) |
